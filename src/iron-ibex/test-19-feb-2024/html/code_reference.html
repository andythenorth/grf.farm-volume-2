Code reference

Obtaining source code

Iron Ibex source code is found on GitHub. License is GPL v2.


Compiling

Iron Ibex source includes a makefile for convenient building. This requires 'Make', typically provided by the ports tree on Linux / Unix, MSYS on Windows and Xcode on macOS.

Other dependencies include:

  • Python 3.10+
  • NML 0.6.0 or newer (NewGRF compiler)
  • Chameleon (Template engine)
  • Markdown (Markdown text format parser)

Railtypes and fallbacks

Iron Ibex uses a very specific approach to railtypes.

  1. The Iron Ibex grf includes both vehicles and railtypes. This is unusual; it's more common to split up vehicles and railtypes to separate grfs. However this puts more work on the player to find an appropriate grf, if they're even aware they need to. Iron Ibex provides batteries-are-included gameplay by including the appropriate railtypes.
  2. Players can turn off the included Iron Ibex railtypes if they wish to via parameter setting (does not affect RAIL and ELRL which are provided as default by OpenTTD).
  3. Iron Ibex uses the Iron Horse schema for vehicle track_type property (which sets the vehicle railtype). This schema makes it possible for railtype authors to detect different Iron Ibex vehicle types and handle them as they see fit. Meanwhile the vehicles will fall back to more common railtypes if the specific Iron Ibex labels are not available.

Iron Ibex railtype scheme details / tips for railtype authors

Railtype labels are 4 chars, composed from the following valid characters: a-z, A-Z, 0-9, _.

Iron Ibex labels, characters 1 and 2 are always IH

Character 3 represents a general 'class' of vehicle

  • A vanilla rail vehicle, non-electric, suitable for RAIL and similar railtypes
  • B vanilla rail vehicle, electric (AC), suitable for ELRL and similar railtypes
  • C vanilla metro vehicle, requires dedicated metro lines, suitable for MTRO and similar railtypes, metro is always assumed to be electrified via 3 or 4 rail system
  • D vanilla narrow gauge vehicle, non-electric, suitable for NGRL and similar railtypes
  • E vanilla narrow gauge vehicle, electric (AC), suitable for ELNG and similar railtypes
  • F vanilla rail vehicle, electric (DC)
  • G vanilla rail vehicle, electric (AC / DC)
  • …more 'classes' added as needed

Character 4 represents modifiers to the class, and is used sparingly. Modifiers cannot be combined, except by adding a new character specifically for the combined properties.

  • _ no modifier used (default)
  • A can be used on high speed lines (LGV), has suitable brakes, signalling equipment etc.
  • …more modifiers added as needed

The uses of labels specific to Iron Ibex means that railtype authors can control precisely how they wish to map Iron Ibex vehicles to railtypes in their railtype grf. This is done using the railtypes property alternative_railtype_list.

For some vehicle types, Iron Ibex may also set labels from the Standardised Railtype Scheme as fallbacks only. Iron Ibex is only partially compliant with the Standardised Railtype Scheme, and uses the "Innsbruck 2022 Convention" where axle load classes are treated as "undefined".
